# DocView — Comprehensive Project Documentation

> **Version**: 2.0 &nbsp;|&nbsp; **Platform**: Android (primary), iOS (secondary) &nbsp;|&nbsp; **Framework**: React Native + Expo SDK 55

---

## 📖 What is DocView?

DocView is a **fully offline, premium document viewer** mobile app built with React Native and Expo. It allows users to open, view, search, and share **30+ file formats** directly on their device — without uploading to any server or requiring an internet connection.

Think of it as a **Google Drive Files viewer clone** but entirely local — every file is parsed, rendered, and stored on-device.

## 🔄 Complete Workflow

### 1. App Startup
```
index.js → Polyfills atob/btoa → Registers App component
    ↓
App.js → Creates NavigationContainer → Wraps in HistoryProvider
    ↓
HistoryProvider → Creates DocView_Docs/ directory → Loads history from AsyncStorage
    ↓
HomeScreen renders with file history list
```

### 2. File Import (User picks a file)
```
User taps FAB (+) → Alert shows "Browse Device" or "Open from URL"
    ↓
DocumentPicker.getDocumentAsync() → Returns { uri, name, size, mimeType }
    ↓
HistoryContext.addToHistory() → Copies file to permanent DocView_Docs/ directory
    ↓
Navigate to ViewerScreen with file object
```

### 3. File Opening in Viewer
```
ViewerScreen receives file → Calls loadFile()
    ↓
1. If remote URL → Download with progress tracking → Cache locally
2. If content:// URI → Copy to cache via react-native-blob-util
3. If no extension → detectExtFromContent() reads magic bytes:
   • %PDF → pdf, PK → check ZIP for Office format
   • 0x89PNG → png, 0xFFD8 → jpg, GIF8 → gif
   • 0xD0CF → doc (OLE2), JSON/HTML detection
    ↓
4. Switch on extension → Route to correct viewer
5. For DOCX/PPTX → Parse ZIP → Extract XML → Convert to HTML → WebViewer
6. For XLSX/XLS → SheetJS parses → Array data → SpreadsheetViewer
7. For text/code → Read as UTF-8 → TextViewer/CodeViewer
```

### 4. Document Rendering Pipeline

#### DOCX Pipeline:
```
Base64 data → JSZip.loadAsync() → Extract word/document.xml
    ↓
Parse relationships (word/_rels/document.xml.rels)
    → Extract image references → Load images as base64 data URIs
    → Extract hyperlink references → Map rel IDs to URLs
    ↓
convertDocxXmlToHtml() → Processes paragraphs & tables in order
    → Headings (h1-h6), bold, italic, underline, strike
    → Colors, font sizes, alignment
    → Embedded images, hyperlinks
    ↓
Rendered HTML → WebViewer component → react-native-webview
```

#### PPTX Pipeline:
```
Base64 data → JSZip.loadAsync() → Find ppt/slides/slide{N}.xml files
    ↓
For each slide:
    → Parse slide relationships (images, hyperlinks)
    → Extract shape tree (p:spTree)
    → Process shapes, pictures, tables, group shapes
    → Extract text with formatting (bold, italic, colors)
    ↓
Build slide cards HTML with slide numbers
    ↓
WebViewer renders styled slides
```

#### Spreadsheet Pipeline:
```
Base64 data → XLSX.read() → Workbook with SheetNames
    ↓
For each sheet → XLSX.utils.sheet_to_json(header: 1)
    → 2D array of cell values (max 200 rows)
    ↓
SpreadsheetViewer → WebView with inline HTML
    → Sticky column headers (A, B, C...)
    → Sticky row numbers (1, 2, 3...)
    → Sheet tab switcher
    → Row/column count info bar
```

### 5. File Persistence System
```
HistoryContext manages all file state:

Storage: AsyncStorage (key: @docview_history_files_v2)
    → JSON array of file entries (max 30 items)
    → Each entry: { uri, originalUri, name, size, mimeType, openedAt, starred }

File Storage: FileSystem.documentDirectory/DocView_Docs/
    → Files copied here persist across app restarts
    → Named: doc_{timestamp}_{sanitized_name}
    → Not affected by OS cache cleanup
```

### 6. File Sharing / Save As
```
Share: expo-sharing → System share sheet (WhatsApp, email, etc.)

Save As (Android):
    → Copy file to Downloads directory
    → MediaCollection.scanFile() → Shows in file manager immediately

Save As (iOS):
    → Copy to cache with custom name
    → Sharing.shareAsync() → "Save to Files" dialog
```

### 7. Intent Handling (Open With)
```
Android: Intent filter in app.json → Matches VIEW/SEND with */* mimeType
    ↓
App.js → Linking.getInitialURL() or addEventListener('url')
    ↓
handleIncomingUrl() → For content:// URIs:
    → ReactNativeBlobUtil.fs.stat() → Get display name
    → Copy to cache directory
    → Detect file type from magic bytes (ZIP → Office format detection)
    ↓
Navigate to ViewerScreen
```

## 🔌 Offline Capability

**DocView is 100% offline.** Every component works without internet:

| Component | How it works offline |
|-----------|---------------------|
| **PDF Viewer** | react-native-pdf renders natively, no network |
| **Image Viewer** | Native Image component, local URIs |
| **Text Viewer** | Pure React Native FlatList, no network |
| **JSON Viewer** | Custom tree renderer, all local |
| **Code Viewer** | Self-contained syntax highlighter in inline JS (no CDN) |
| **Markdown Viewer** | react-native-markdown-display, fully local |
| **Spreadsheet Viewer** | WebView with inline HTML/CSS/JS, no external resources |
| **Web Viewer** | WebView with inline HTML, local-only rendering |
| **DOCX/PPTX Parser** | JSZip + custom XML → HTML conversion, all local |
| **DOC/PPT Parser** | CFB library parses binary, all local |
| **Excel Parser** | SheetJS reads base64 directly, no network |
| **File Storage** | AsyncStorage + expo-file-system, all local |
| **File Picker** | expo-document-picker, system-level |

> **Note**: The "Open from Web URL" feature is the only feature that requires internet — and only at the moment of download. Once downloaded, the file is cached locally.

## 🚀 How to Build and Test

### Prerequisites

1. **Node.js** ≥ 18 installed
2. **Java Development Kit (JDK)** 17
3. **Android SDK** (via Android Studio) or set `ANDROID_HOME` environment variable
4. Git

### Local Development

```bash
# 1. Install dependencies
cd DocView
npm install

# 2. Generate native Android project (if not already present)
npx expo prebuild --platform android

# 3. Start Metro bundler
npx expo start

# 4. Run on connected Android device/emulator
npx expo run:android
```

### Build APK Locally (Debug)

```bash
# Navigate to android directory
cd android

# Build debug APK
./gradlew assembleDebug

# APK output location:
# android/app/build/outputs/apk/debug/app-debug.apk
```

### Build APK Locally (Release)

```bash
cd android
./gradlew assembleRelease

# APK output:
# android/app/build/outputs/apk/release/app-release.apk
```

## ☁️ Building with EAS (Expo Application Services) — Cloud Build

EAS Build lets you build the APK/AAB in the cloud without local Android SDK setup.

### Step 1: Install EAS CLI
```bash
npm install -g eas-cli
```

### Step 2: Log in to Expo
```bash
eas login
# Create an account at https://expo.dev if you don't have one
```

### Step 3: Configure EAS Build

Create an `eas.json` in your project root (if not already present):

```json
{
  "cli": {
    "version": ">= 16.0.0"
  },
  "build": {
    "development": {
      "developmentClient": true,
      "distribution": "internal",
      "android": {
        "buildType": "apk"
      }
    },
    "preview": {
      "distribution": "internal",
      "android": {
        "buildType": "apk"
      }
    },
    "production": {
      "android": {
        "buildType": "apk"
      }
    }
  }
}
```

### Step 4: Build

```bash
# Development build (debug APK with dev client)
eas build --platform android --profile development

# Preview build (release APK for testing)
eas build --platform android --profile preview

# Production build (release APK for distribution)
eas build --platform android --profile production
```

### Step 5: Download & Install

After the cloud build completes:
1. EAS provides a **download link** for the APK
2. Download the APK to your phone
3. Open the APK file → Install (you may need to enable "Install from unknown sources" in Settings → Security)

## 🔒 Security & Privacy

- **No data transmission**: All files stay on-device. Zero network calls for file processing.
- **No analytics**: No tracking, telemetry, or crash reporting.
- **No cloud storage**: Files are stored in the app's sandboxed document directory.
- **Permissions**: Only `READ_EXTERNAL_STORAGE` (for accessing device files).

---

## ⚡ Performance Optimizations

- **FlatList virtualization**: TextViewer uses FlatList with `getItemLayout` for instant scrolling
- **Lazy loading**: Viewers only parse files when opened
- **Cache management**: Content URI cache limited to 20 entries, auto-cleanup of files > 1 hour old
- **Sheet row limits**: Spreadsheets capped at 200 rows to prevent OOM
- **Memoized rendering**: JSON tree and text lines use React.memo / useCallback
- **Binary detection**: Magic byte detection avoids parsing wrong formats

## 🐛 Known Limitations

| Limitation | Details |
|-----------|---------|
| **Legacy .doc** | Text-only extraction; no formatting/images (binary OLE2 is complex) |
| **Legacy .ppt** | Text extraction works; images not extracted from binary format |
| **Large PDFs** | Very large PDFs (100+ MB) may be slow on low-end devices |
| **Spreadsheet rows** | Limited to first 200 rows for performance |
| **History limit** | Max 30 files in history (configurable in HistoryContext.js) |
| **Code highlighting** | Keyword-based (not full AST); good for reading, not IDE-level |
